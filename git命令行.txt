6.推送到远程 git push origin main

问题：Git 通过 HTTPS 推送时网络被重置，无法连接服务器。
解决：改用 SSH 协议，生成密钥并添加到 GitHub。

1.ssh-keygen -t ed25519 -C "你的GitHub邮箱@example.com"

2.cat ~/.ssh/id_ed25519.pub

3.复制屏幕上显示一大串以 ssh-ed25519 开头的字符，这就是公钥
打开浏览器登录你的 GitHub 账户。点击右上角头像 -> Settings。在左侧菜单栏找到并点击 SSH and GPG keys。
点击绿色的 New SSH key 按钮。
在 Title 栏可以随意填一个名字（比如 "My Windows PC"），然后在 Key 栏粘贴你上一步复制的公钥内容。
点击 Add SSH key 完成。

4.回到 Git Bash 终端：测试一下和 GitHub 的连接是否成功：
   ssh -T git@github.com
   如果看到 Hi yefe1993! You've successfully authenticated... 的字样，说明配置成功了。

现在可以推送代码了：
git push origin main
