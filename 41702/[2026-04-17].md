# [2026-04-17]

### 🕳 现象描述

点击 QML 按钮 **[ 归并排序 ]** 触发 `mS()`：

- 按钮无反应，程序卡死
- 按钮有反应，但 `sortDisplay` 显示 `NULL`

### 👁 问题定位

**① `split()` 未断链 → 链表结构被污染**

```C++
ListCtr::Node *ListCtr::split(Node *h, int i)
{
    Node* rH = h;

    for(int j = 1;j < i && rH != nullptr;++j){
  ❌    rH = rH->next;                           
  ❌}                                           
  ❌return rH->next;                             
}
```

- 只“分段”，没有“断开”，子链表尾next指针未置空
- 导致多个子链共享同一段内存，后续 `merge` 时发生链表交叉、覆盖 



**② `prev` 移动逻辑错误 → 指针走到虚空**

```C++
ListCtr::Node *ListCtr::mergeSort()
{
    ...
    while(curr){
            ...
          ❌while(prev){
          ✅while(prev->next){ // 停在尾节点，而不是 nullptr
                  prev = prev->next; 
            }
          ❌curr = prev->next; // 此时 prev->next == nullptr
    } //curr 空指针访问，内层循环提前终止
    ...
}
```



**③ `dummy.next` 未初始化 → 野指针风险**

```C++
Node dummy(0);  // 缺少 dummy.next = head;
```

导致：`mergeSort()` 起点不确定 + 结果不可预测



**④ 外层循环未重置 `curr`**

```C++
ListCtr::Node *ListCtr::mergeSort()
{
    ...
  ❌Node* curr = head;

  ❌while(i < length/2){ 
  ✅while(i < length){ // 归并排序必须跑到覆盖整个链表
        ...
		✅ Node* curr = dummy.next;
        while(curr){
            ...
            curr = next;
        }
        i = i * 2;
    }
	...
}
```

但每轮 `i *= 2` 后，没有重新从链表头开始，导致 后续轮次 根本没执行 完整排序

> **结论：**链表在排序过程中没有被“正确切割 + 正确拼接”，核心指针逻辑失控，导致结构崩坏。
