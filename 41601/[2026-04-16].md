### 👁 问题定位

```C++
// ListCtr.h
// ❌ 错误设计：把“操作函数”当成“属性读取函数”
Q_OBJECT
Q_PROPERTY(QString displayShow READ displayShow CONSTANT)

Q_PROPERTY(QString pushF READ pushF NOTIFY pushFChanged)
Q_PROPERTY(QString pushB READ pushB NOTIFY pushBChanged)
Q_PROPERTY(QString popF READ popF NOTIFY popFChanged)
Q_PROPERTY(QString popB READ popB NOTIFY popBChanged)
    
...
    
public:
Q_INVOKABLE void pushF(int val);
Q_INVOKABLE void pushB(int val);
Q_INVOKABLE void popF();
Q_INVOKABLE void popB();

signals:
void pushFChanged();
void pushBChanged();
void popFChanged();
void popBChanged();
```

**错误地将“链表操作函数”声明为 Q_PROPERTY 的 READ 函数**，导致：

- 类型不匹配（`QString` vs `void`）
- moc 代码生成异常
- 最终引发编译错误甚至崩溃

### 🧠 根因分析

没建立**Q_PROPERTY** 和 **Q_INVOKABLE** 的规则模型

```
 【职责模型图】
 		QML界面
           ↓
   ┌───────────────┐
   │ Q_PROPERTY    │  ← 数据（给界面看）
   └───────────────┘
           ↑
     signal（NOTIFY）
           ↑
   ┌───────────────┐
   │ Q_INVOKABLE   │  ← 操作（用户触发）
   └───────────────┘
```

|       维度       |               Q_PROPERTY （属性）               |     Q_INVOKABLE（可调用的）     |
| :--------------: | :---------------------------------------------: | :-----------------------------: |
|       本质       |                  数据 （只读）                  |         操作 （读/写）          |
|       用途       |              给 QML **读取/显示**               |       给 QML **调用执行**       |
|       规则       | `Q_PROPERTY(type name READ func NOTIFY signal)` | `Q_INVOKABLE void name(params)` |
|     函数要求     |              `type func()`（无参）              |       `void func(params)`       |
| 是否触发 UI 更新 |               通过 `NOTIFY` 信号                |      调用后手动触发 signal      |
|   是否参与绑定   |  支持绑定（自动更新 **NOTIFY** 需要 signal ）   |                /                |
|   QML典型场景    |           显示文本   `text: obj.xxx`            |      按钮点击  `obj.xxx()`      |

### 🔥 实际解决路径

```C++
Q_OBJECT
Q_PROPERTY(QString displayShow READ displayShow NOTIFY displayChanged)

public:
QString displayShow() const;

// QML 调用
Q_INVOKABLE void pushF(int val);
Q_INVOKABLE void pushB(int val);
Q_INVOKABLE void popF();
Q_INVOKABLE void popB();

signals:
//  NOTIFY 信号
void displayChanged();
```

### 数据流步骤总结

| 步骤 |   层级    |            动作             |     数据形式     |
| :--: | :-------: | :-------------------------: | :--------------: |
|  ①   |    QML    | 用户输入，`parseInt()` 转换 | `string` → `int` |
|  ②   | QML → C++ |   调用 `Q_INVOKABLE` 方法   |    跨边界调用    |
|  ③   |    C++    |    操作链表（增删节点）     |     指针操作     |
|  ④   |    C++    |   `emit displayChanged()`   |     信号发射     |
|  ⑤   | C++ → QML | `displayShow()` 返回字符串  |    `QString`     |
|  ⑥   |    QML    |      属性绑定自动更新       |     界面刷新     |
|  ⑦   |   用户    |          看到结果           |     视觉反馈     |
